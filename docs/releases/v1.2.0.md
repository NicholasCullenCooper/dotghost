# dotghost v1.2.0

This release turns dotghost from a basic single-file symlink helper into a typed registry tool that can mount the right workflow surface for each repository.

## Highlights

- TypeScript CLI rewrite with modular source files under `src/`
- selective mounting with registry-relative globs and `--exclude`
- `.dotghostignore` support for registry-level default hidden content
- named profiles through `dotghost.profiles.json`
- `dotghost profiles` discovery command
- profile-aware help text and typo suggestions

## Practical impact

- a shared registry can now expose compact named surfaces such as `core`, `onboarding`, or `delivery`
- registries can keep candidate or experimental content nearby without mounting it by default
- the CLI is easier to extend safely because command parsing, matching, runtime logic, and profile parsing are now separated

## Commands to know

```bash
dotghost mount --profile core
dotghost mount "AGENTS.md" "workflows/**"
dotghost mount --exclude "skills/**"
dotghost mount --include-ignored "candidate/**"
dotghost profiles
```

## Notes

- control-plane files such as `.dotghostignore` and `dotghost.profiles.json` are never mounted into target repositories
- common OS metadata files are ignored during registry traversal
- plain `dotghost mount` still mounts the full visible registry surface, but now points users at a recommended profile when one exists
