# dotghost v1.3.0

This release extends registry profiles from simple named selections into reusable building blocks.

## Highlights

- added `extends` support to `dotghost.profiles.json`
- supports single-parent and multi-parent profile composition
- detects profile cycles and unknown parent references with clear errors
- keeps composed include and exclude lists deduplicated in parent-first order

## Example

```json
{
  "shared": {
    "include": ["AGENTS.md", "CLAUDE.md"]
  },
  "delivery": {
    "extends": "shared",
    "include": ["commands/**", "workflows/**", "skills/testing/**"]
  }
}
```

## Practical impact

- registries can keep a small base profile for shared root files
- user-facing profiles can stay smaller and easier to review
- composition removes duplication in larger registries without changing the mount UX

## Notes

- child profiles are resolved after all parent profiles
- cycles and missing parent names are treated as configuration errors
- first-party `dotagent` profiles now use composition as the reference model
